# React Study Policy

## Purpose

- React / TypeScript / TSX を「読める」だけでなく、「書ける」「説明できる」状態に近づける
- サンプルアプリの実コードを教材にして、処理の流れをつなげて理解する
- 曖昧な理解のまま先へ進まず、足元を固めながら定着させる

## Daily Study Style

- 基本は `1日30分`
- 毎日少しずつ反復する
- 学習は `読む / 書く / 説明する` の3軸で進める
- コードリーディング本編とは分けて、この場では定着と再現力を重視する

## Quiz Policy

- 小テストは基本 `2〜3問`
- まずは `書く1問` `説明する1問` `接続または流れ1問` を基本セットにする
- 四択は補助としてのみ使い、基本は短い記述式と穴埋め式を優先する
- 短くても考える必要がある問題にする

## Order Of Study

- 基本は処理の流れに沿って順番に進める
- 優先順:
  - `index.html`
  - `frontend/src/main.tsx`
  - `App()`
  - `frontend/src/pages/AssetListPage.tsx`
  - `frontend/src/lib/api.ts`
- `api.ts` や `types` は、必要になったタイミングで接続先として扱う
- ランダム出題は補助的に行い、基本は入口から順に積み上げる

## Mastery Rule

- 曖昧な理解のまま次へ進みすぎない
- ただし `完璧` を待ちすぎて止まりすぎない
- 次へ進む目安:
  - 自力で短く書ける
  - 短く説明できる
  - 1つ前のファイルとのつながりを言える
- 同じ論点で `2回連続ほぼ正解` を一つの目安にする
- 怪しかった論点は翌日以降に再出題する

## Review Rule

- 新しい範囲に進んでも、前に学習した範囲を毎回少し混ぜる
- `未理解` `要再確認` `定着気味` の感覚で進捗を見る
- 進捗の見た目より、再現できる理解を優先する

## Files To Read Before Each Quiz

- この `study-policy.md`
- `work/frontend-study-log/` にあるこれまでの学習記録
- 特に直近のログと、そこで曖昧だった論点

## How To Prepare The Next Quiz

- 前回までのログを読んで、怪しかった論点を確認する
- その日の対象ファイルを1つ決める
- そのファイルについて `書く / 説明する / 接続または流れ` の順で問題を作る
- 必要に応じて、前回の苦手論点を1問目に再出題する
