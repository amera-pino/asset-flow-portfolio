# Daily Quiz Prompt

以下の手順で、今日の React 復習テストを開始してください。

## Instructions

1. まず `/Users/nomataito/Code/asset-flow/work/study-log/study-policy.md` を読んで、学習方針と出題方針を確認してください。
2. 次に `/Users/nomataito/Code/asset-flow/work/study-log/study-log-template.md` を確認し、日次ログの基本構成を把握してください。
3. そのあと `/Users/nomataito/Code/asset-flow/work/study-log/` 配下の学習ログを確認してください。
4. 特に直近のログを優先して読み、前日までに出題した内容、正答状況、曖昧だった論点を把握してください。
5. その内容を踏まえて、今日のテストで扱う対象ファイルと論点を決めてください。
6. 出題は基本的に処理の流れ順で進めてください。
   - `index.html`
   - `frontend/src/main.tsx`
   - `App()`
   - `frontend/src/pages/AssetListPage.tsx`
   - `frontend/src/lib/api.ts`
7. ただし、前回までに曖昧だった論点があれば、新しい範囲に進む前に優先して再出題してください。
8. テストは `30分` を目安に、基本 `2〜3問` で構成してください。
9. 問題形式は基本的に以下を優先してください。
   - `書く問題`
   - `説明する問題`
   - `接続または流れの問題`
10. 四択は補助としてのみ使い、短い記述式や穴埋め式を優先してください。
11. 出題前に、今日は何を確認する回かを短く宣言してから問題を出してください。
12. 必要に応じて、`getElementById` や `render` のような重要キーワードを穴埋め問題として出してください。

## Output Style

- まず、今日確認した `study-policy.md` と直近ログの要点を1〜3行で要約してください。
- 次に、今日のテスト対象と狙いを短く示してください。
- そのあと、問題を順番に出してください。
- 回答後は、事前に決めたフィードバック方針に従って返してください。
- 学習ログを新規作成するときは、`study-log-template.md` をベースにして記録してください。
- 穴埋め問題は、実装で重要なキーワードを反射的に書けるようにするために混ぜてください。

## Notes

- 曖昧な理解のまま先へ進みすぎない
- ただし完璧主義で止まりすぎず、`自力で書ける / 説明できる / つながりを言える` を目安にする
- 新しい範囲に進んでも、必要なら前の範囲を混ぜて再確認する
