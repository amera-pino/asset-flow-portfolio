## Understood

- `lifespan` は `async def` で定義された関数で、起動時と終了時の処理をまとめて書くために使う
- `@asynccontextmanager` は、その関数を `async with` で扱えるコンテキストマネージャーにするためのデコレータ
- `yield` の前は起動前処理で、`yield` に到達すると FastAPI に制御が渡り、リクエスト受付が始まる
- `settings = get_settings()` により、`config.py` で定義した設定インスタンスを受け取る
- `FastAPI(...)` は FastAPI アプリのインスタンス生成で、`title` はドキュメント表示名、`lifespan` は起動・終了処理の受け渡し
- `CORSMiddleware` は別オリジンからのアクセスを制御する
- `allow_origins` は許可するURLのリストを渡す
- `allow_credentials` は Cookie や認証情報を扱う設定
- `allow_methods` は許可する HTTP メソッドの設定
- `allow_headers` は許可するヘッダーの設定
- `include_router(..., prefix="/api")` はルーターをアプリに登録し、URL 先頭に `/api` を付ける

## Still Fuzzy

- `yield` の説明は、`一時停止` `制御を渡す` `受付開始` の言い換えがまだ混ざりやすい
- `@asynccontextmanager` と `lifespan` の関係を、関数・コンテキストマネージャーの両面から安定して言い分けたい
- `CORSMiddleware` の各設定値は理解できてきたが、自然に即答できるよう反復が必要
- `include_router()` の説明で、`登録する` と `配下にまとめる` の表現を安定させたい
