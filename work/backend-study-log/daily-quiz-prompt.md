# Daily Quiz Prompt

以下の手順で、今日の Backend 復習テストを開始してください。

## Instructions

1. まず `/Users/nomataito/Code/asset-flow/work/backend-study-log/study-policy.md` を読んで、学習方針と出題方針を確認してください。
2. 次に `/Users/nomataito/Code/asset-flow/work/backend-study-log/study-log-template.md` を確認し、日次ログの基本構成を把握してください。
3. そのあと `/Users/nomataito/Code/asset-flow/work/backend-study-log/` 配下の学習ログを確認してください。
4. 特に直近のログを優先して読み、前日までに出題した内容、正答状況、曖昧だった論点を把握してください。
5. その内容を踏まえて、今日のテストで扱う対象ファイルと論点を決めてください。
6. 出題は基本的に処理の流れ順で進めてください。
   - `backend/app/main.py`
   - `backend/app/core/config.py`
   - `backend/app/core/database.py`
7. ただし、前回までに曖昧だった論点があれば、新しい範囲に進む前に優先して再出題してください。
8. 1つの関数や処理は、1回で終わらせず、複数の角度から反復して出題してください。
9. 問題は `1問ずつ` 出してください。ユーザーが解答したら、まず `正解 / 不正解` を伝え、そのあとに短く解説してから次の1問に進んでください。
10. 最初は難易度を下げて、`穴埋め` や `重要キーワードの役割確認` から始めてください。
11. 慣れてきたら、`短い記述` `流れの説明` `コードの一部を書く問題` の順で少しずつ難易度を上げてください。
12. 四択は補助としてのみ使い、短い記述式や穴埋め式を優先してください。
13. いきなり全文を書かせず、重要箇所の一部だけを書かせる問題から始めてください。
14. テストは `30分` を目安に、結果として `2〜3問` ほど扱う想定で進めてください。
15. 問題形式は基本的に以下を優先してください。
   - `書く問題`
   - `説明する問題`
   - `接続または流れの問題`
16. 出題前に、今日は何を確認する回かを短く宣言してから問題を出してください。
17. 必要に応じて、`FastAPI` や `create_engine`、`get_settings` のような重要キーワードを穴埋め問題として出してください。
18. ユーザーが途中で「一旦ストップ」や「次に行かないで」と言った場合は、その論点に戻って反復してください。

## Output Style

- まず、今日確認した `study-policy.md` と直近ログの要点を1〜3行で要約してください。
- 次に、今日のテスト対象と狙いを短く示してください。
- そのあと、問題を順番に出してください。
- 回答後は、事前に決めたフィードバック方針に従って返してください。
- 学習ログを新規作成するときは、`study-log-template.md` をベースにして記録してください。
- 穴埋め問題は、実装で重要なキーワードを反射的に書けるようにするために混ぜてください。
- 問題ごとのフィードバックは、`正解 / 不正解` を先に短く返し、そのあとに説明してください。

## Notes

- 曖昧な理解のまま先へ進みすぎない
- ただし完璧主義で止まりすぎず、`自力で書ける / 説明できる / つながりを言える` を目安にする
- 新しい範囲に進んでも、必要なら前の範囲を混ぜて再確認する
